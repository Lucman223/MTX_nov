Arquitectura Front End Back End Frameworks Negocio

MotoTX está construido bajo una filosofía de separación de responsabilidades, garantizando que el núcleo de negocio sea independiente de la interfaz de usuario.

Esta arquitectura permite una escalabilidad horizontal masiva. El Frontend actúa como un cliente agnóstico que consume una REST API protegida. Además, implementamos un flujo de **Despliegue Continuo (CD)** automatizado hacia Koyeb, garantizando que cada mejora técnica se valide y despliegue en minutos.

⚙️ Bloque 2: Back End y Protección de Datos

El motor de persistencia utiliza Eloquent ORM para garantizar transacciones seguras. La lógica se centraliza en "Services" de PHP 8.2+, desacoplando los controladores de la complejidad del negocio y facilitando las pruebas unitarias.

2.1 Lógica de Solicitud de Viaje (Calculo y Validación)

ViajeService.php Laravel Service

<?php
public function solicitarViaje(User $user, $data): Viaje {
    // Verificación de saldo (Forfaits)
    $clienteForfait = ClienteForfait::where('cliente_id', $user->id)
        ->where('viajes_restantes', '>', 0)
        ->first();

    if (!$clienteForfait) throw new \Exception('No active forfait.');

    $viaje = Viaje::create($data); // Creación del recurso
    $clienteForfait->decrement('viajes_restantes'); // Actualización de cupo
    return $viaje;
}

const pollStatus = async (orderId) => {
    const interval = setInterval(async () => {
        const res = await axios.post('/api/forfaits/check-status', { order_id: orderId });
        if (res.data.status === 'SUCCESS') finishPurchase();
    }, 2000);
};
